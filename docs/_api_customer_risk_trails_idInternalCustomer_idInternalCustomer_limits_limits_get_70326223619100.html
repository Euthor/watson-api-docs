GET /api/customer/risk-trails?idInternalCustomer=:idInternalCustomer&limits=:limits

Description

Read the Risk Trails for a given Customer. Note that the last Risk Trail also contains the current approval status for the Customer.

Headers

Content-Type

application/json

x-access-token

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9

Parameters

idInternalCustomer required, type: string

Declare in query string. The Internal Customer Number of the requested Customer.

limits required, type: integer

Declare in query string. How many Risk Trails to fetch. Note that Risk Trails are fetched in descending creation order so limits=1 will fetch the last Risk Trail. If the last Risk Trail has idApprovalStatus=null then the Customer Approval Status is 'Pending'

Example Response

{
    "id": 1,
    "idRiskTrail": 1,
    "reviewed": false,
    "reviewedDate": null,
    "comments": null,
    "idCustomer": 1,
    "idRiskType": null,
    "idUserReviewer": null,
    "userReviewerName": null,
    "idApprovalStatus": null,
    "approvalStatusName": null
}

REST API v.1.0

Login

These endpoints belong to the login resource.
Resource Description
POST /api/login Login using username & password and receive an access token to be used as an x-access-token header in subsequent requests. Ensure the requested user has the appropriate priviledges for subsequent requests.
Customer

These endpoints belong to the customer resource.
Resource Description
GET /api/customer/:idInternalCustomer Read a Customer by his Internal Customer Number
POST /api/customer Create a new Customer. Responds with the unique identifiers of the created Risk and Audit Trails.
PUT /api/customer/:idInternalCustomer Update an existing Customer. Responds with the unique identifiers of the created Risk and Audit Trails.
DELETE /api/customer/:idInternalCustomer Delete a Customer from the system.
Risk analysis

These endpoints belong to the Risk Analysis resource.
Resource Description
GET /api/customer/:idInternalCustomer/risk Get a Risk Analysis, also known as Calculated Risk, for a given Customer.
Risk trail

These endpoints belong to the Risk Trail resource.
Resource Description
GET /api/customer/:idInternalCustomer/risk-trails?limits=:limits Read the Risk Trails for a given Customer. Note that the last Risk Trail also contains the current approval status for the Customer.
PUT /api/customer/risk-trails/:idRiskTrail Update a Risk Trail. Used to approve a Risk Trail.
